Lambda Expressions(Behavior Parameterization)

Lambda表达式是解决行为参数的优雅方案。传统的Strategy Pattern甚至是匿名类充斥着冗余代码，而Lambda表达式只需要关注真正行为。A functional interface is any interface that contains only one abstract method。Lambda表达式便可作为参数传给这种Functional Interface。

@FunctionalInterface 注解只是为了表明这是一个Functional Interface，如果提供多个abstract method，会引导编译器报错。

Lambda对本地变量的引用必须是final或者是effectively final，openjdk文档里说这样主要是为了禁止在lambda表达式中修改影响外部变量，尤其是在lambda可能用在不同的线程中，本地变量又是存在当前线程栈内。

Java并发编程实战(Java Concurrency in Practice)

发表于 2017-07-20 | 分类于 读书笔记

————Java Concurrency in Practice, Java 并发必读经典(最好结合英文原版一起阅读，中文版翻译实在晦涩，并且本书很多内容点到为止并没有深度展开讨论)

基础知识

线程安全性

  • 所谓线程安全，指的是对象在多线程环境执行下的行为依然符合预期。
  • 无状态的对象一定是线程安全的。
  • 如果一个对象的状态会在多线程环境下被读取/更新，通常使用内置的线程安全类来承载这种状态是合理的做法。使用这种内置安全类提供的原子操作来完成更新动作。如：AtomicLong。
  • synchronized block可以对代码块加锁，在函数前面加synchronized相当于synchronized(this)。也有static synchronized对类加锁，类的所有实例互斥访问该代码块。[TIPS]: 一个类的多个不同synchronized函数也是互斥的。
  • Reentryancy指的是一个线程可以多次获得已获得的锁(类似计数的方法)，避免子类同步函数访问父类同步函数造成死锁。
  • 除了synchronized整个函数，有时候需要权衡简单性和并发性，在函数内控制更细粒度的加锁块。

对象的共享

  • 可见性，volatile修饰的变量能确保其修改在其他线程中及时看到，这将影响这编译器的重排，数据的更新也会及时写入内存而不会缓存在寄存器。
  • 线程封闭技术实际上并不是共享对象，而是确保对象只能在一个线程中使用。ThreadLocal方法就是通过在每个线程中单独创建对象实现封闭。

对象的组合

  • 将对象委托给现有线程安全类，是一种常用的获得线程安全类的手段

基础构建模块

  • 旧的Vector、Hashtable、Collections.synchronizedXxx等同步容器类，基本已经失去了使用价值。
  • 并发容器
    • ConcurrentHashMap: 采用分段锁提高吞吐
    • CopyOnWriteArrayList/Set: 每次修改都会复制并替换底层数组来确保旧引用的不可变，用于类似订阅/通知读多改少的业务
    • BlockingQueue/Deque: 阻塞队列
  • 同步工具类
    • CountDownLatch
    • FutureTask: 异步任务
    • Semaphore: 计数信号量
    • CyclicBarrier/Exchanger
  • [TIPS]: 最终用ConcurrentHashMap + FutureTask + Double-Checked实现了一个简易缓存。业界通常使用Guava的LoadingCache来做内存缓存了。

使并发应用结构化

任务执行

  • Executor框架
    • Executors线程池
    • Scheduled定时延迟任务
  • Runnable、Callable、Future的配合

取消与关闭

  • 守护线程

线程池的使用

  • 避免在线程池中的线程继续提交线程任务，容易造成线程饥饿死锁
  • 管理任务队列(无界、有界、同步队列)
  • 提供饱和策略(Abort、Caller-Runs、Discard、Discard-Oldest、[TIPS]: 自定义的阻塞策略)
  • 线程工厂

Liveness、性能与测试

  • 死锁
  • 饥饿
  • 活锁

Performance and Scalability

  • 性能指的程序运行的快慢，可伸缩指的当计算相关资源变化时，程序的处理能力也要随之变化
  • 线程引入的开销
    • 上下文切换，微秒级别的(vmstat查看线程切换所占比例从而评估程序状态)
    • 内存同步，共享资源在线程间的内存同步。(现代JVM对加锁代码的优化已经可以很好的去掉非竞争情况下的开销了)
    • 阻塞，因为同步带来的线程阻塞切换。(现代JVM也会有如自旋锁来优化这种情况)
  • 减少锁竞争
    • 缩小锁范围
    • 减小锁粒度
    • 分段锁
    • 避免热点域(如所有操作都需要维护一个计数器，这样一个计数器在并发环境下就是热点域。[TIPS]: HashMap通过维护计数器在调用size时直接返回，而ConcurrentHashMap则只能通过分段计数，最后累加输出)
  • 减少上下文切换的开销
    • 日志输出通过集中在单独一个线程输出要比在业务线程输出效率高，避免了I/O竞争导致频繁的线程切换

高级主题

显示锁

  • ReentrantLock implements Lock: ReentrantLock实现了Lock接口，提供了灵活的轮询、定时、中断等功能
  • ReentrantReadWriteLock implements ReadWriteLock: 读写锁将锁分为读写两部分，以供读多写少或读写不同情况使用
  • 锁的公平性、读写锁的升级降级等各种优化都是为了让程序并发更顺畅和迅速，减少线程上下文开销

Others

  • Java 8 AtomicLong vs LongAdder vs LongAccumulator: 核心概念CAS，Cells
  • Java 8 ConcorrentHashMap: Java 7 分段锁，size做法，Java 8 引入红黑树
  • Java 8 StampedLock: 在极端的读多写少情况下，读取可以使用乐观的方式，读取前后使用stamp确保没有写操作。如果不巧遇到了写操作，可以使用悲观读继续操作。

Linear Regression

线性回归的数学表达，Cost Function使用梯度下降求解的空间意义。

Logistic Regression

逻辑回归通常用于分类问题，0/1分类问题常使用sigmod函数将输出域映射到[0, 1]，进而可以表达接近0或1的概率。其Cost Function为分段函数，空间意义依然可以使用梯度下降求解。

多分类的逻辑回归依然可以映射得出每个类的概率，概率和为1。使用softmax函数输出。

Nerual Network

神经网络模型通过多层的参数学习，一层层的线性关系最终构成复杂的关系。以图像识别常用的卷积神经网络为例，经过多层的卷积以及feature提取，每一层学到的通常刚好能够代表要识别内容的特征，一层层传递加强，最终得到分类概率分布。
